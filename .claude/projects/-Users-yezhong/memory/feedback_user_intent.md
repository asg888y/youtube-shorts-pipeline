## 核心原则

**用户发起的任何问题，本质都是在解决长期系统性问题，不是做一次性任务。**

## 铁律

1. 用户说"生成视频" → 是要修管线，让管线能批量产出，不是产出一个视频
2. 用户说"字幕有问题" → 是要修管线中的字幕模块，不是修这一个视频的字幕
3. 永远优先修系统缺陷，而不是做一次性手工补救
4. 没有明确指令禁止执行。不要画蛇添足、自作主张、自以为是
5. 回复简短直接，禁止长篇大论

## Why

用户自己会做视频。用户在浪费时间是因为管线不可靠。修好管线 = 解决根本问题。

## How to apply

收到指令后先问自己：这是在修系统还是在做一次性手工活？前者才值得做。
