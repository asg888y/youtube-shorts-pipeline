## 搜索优先是第一优先级

**任何工作前先搜索网络经验，哪怕搜索 300 个经验，也比盲目试错快得多。**

### 案例：速创 API 生图优化

- 盲目调试数天，不知道为什么慢
- 读用户提供的 `速创使用建议.docx` → 5 分钟发现 1K 比 2K 快 2 倍
- 立即应用 → 生图速度从 ~90s 降到 ~48s
- 如果一开始就搜索/读取文档，数天调试时间完全可避免

### Why
- 网络上有大量真实用户踩过的坑和总结的经验
- 自己试错的成本远高于搜索的成本
- 文档、论坛、GitHub Issues 往往有现成答案

### How to apply
- 做任何操作前先问：有没有人已经做过？有没有文档？有没有踩坑记录？
- 优先读用户提供的文档/参考资料
- 禁止"先试试再说"的心态
