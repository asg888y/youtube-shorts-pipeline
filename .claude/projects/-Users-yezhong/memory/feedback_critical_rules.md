## 铁律

1. **做任何创作/修复前，先读 MEMORY.md 和相关 feedback 记忆**。不读记忆就动手 = 必然重复犯错。
2. **API/Key 失效立即跳过**。不要调试 API，用已有资源继续。上限 1 次尝试。
3. **禁止反复调试同一个问题**。一个错误修 1 次还不行 → 停下来分析根因，不要反复改同一个参数。
4. **做完立即保存工作结果到记忆**。配置修改、成功参数、失败教训，全部落盘。
5. **严格按用户指令执行，不要自行扩展创作范围**。用户说生产 1 个视频就 1 个，不要加戏。
6. **遇到超过 5 分钟无进展的问题 → 切换方案**，不要死磕。

## 为什么

用户反复看到同样的错误（字幕叠层、字号过大）被"修复"多次，因为每次会话都没有从记忆中学到上一轮的教训。Token 被浪费在重复调试上。

## 执行

- 任务开始：读 MEMORY.md → 读相关 feedback → 再动手
- 任务结束：关键参数/配置/教训 → 写 feedback memory → 更新 MEMORY.md
