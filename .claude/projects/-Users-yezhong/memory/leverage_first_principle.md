# 先利用再创造（Leverage First, Then Create）

**核心原则**：做任何功能前，先问"有没有现成工具可以做到？"而非"我该怎么写代码实现？"

**Why**: Pixelle-Video 案例验证 — 我们花了大量时间自己写 pipeline、API client、渲染逻辑，而 AIDC-AI/Pixelle-Video 已有完整的 8 步流水线、29 个工作流、31 个模板、6 个 LLM 预设、双模式 TTS、REST API + Web UI。自定义现成工具只花了原来 1/10 的工作量。

**How to apply**:
1. 遇到需求 → 先搜 50+ 网络资源找现成工具
2. 找到工具 → 先完整理解其架构和可替换点
3. 映射需求 → 哪些模块直接可用、哪些需配置、哪些需替换
4. 替换而非重写 — 在工具的插槽里换上自己的东西
5. 只在无可替代时才自己创造

**检查清单**：
- [ ] 这个功能是否有开源工具已实现？
- [ ] 这个工具的哪些模块可以配置/替换？
- [ ] 我们的哪些组件可以接入工具的替换点？
- [ ] 哪些部分真正需要自己写？
