**Verticals v3** 是开源项目 `youtube-shorts-pipeline` 的第三代版本，由 Dr Rushindra Sinha 开发，现更名为 **Verticals**，是一款**AI 原生的竖屏短视频全自动生产引擎**。它采用 MIT 协议完全开源免费，核心定位是"Topic in, Published Short out"——输入一个话题，自动完成调研、写稿、生成画面、配音、字幕、配乐、缩略图到上传的全流程。

## 一、完整功能清单

| 模块 | 功能说明 | 技术实现 |
|------|---------|---------|
| **Niche Intelligence** | 领域智能配置文件，控制全链路风格 | YAML 配置文件，15+ 内置领域 |
| **Research 调研** | 基于 DuckDuckGo 实时搜索，抓取事实防幻觉 | 网页抓取 + 边界截断（300字符） |
| **Script 脚本** | 生成 60-90 秒钩子驱动旁白脚本 | 多 LLM 支持，含 B-roll 提示词、缩略图提示词 |
| **Visuals 视觉素材** | 生成 3-5 张 9:16 B-roll 画面 | Gemini Imagen / Replicate / Pexels / **ComfyUI** |
| **Voice 语音** | 自然语音旁白合成 | Edge TTS / ElevenLabs / **Kokoro(本地)** / macOS `say` |
| **Captions 字幕** | 词级时间戳高亮字幕 | Whisper，输出 ASS（烧录）+ SRT（上传） |
| **Assemble 组装** | 视频最终合成 | ffmpeg + Ken Burns 缩放平移 + 背景音乐 + 自动语音闪避 |
| **Thumbnail 缩略图** | 平台封面图生成 | AI 生成 + 文字叠加 |
| **Upload 上传** | 自动发布 | YouTube（默认私密），TikTok/Reels/X 计划 v3.1 支持 |
| **Topic Discovery** | 热点话题自动发现 | Reddit / RSS / Google Trends / Twitter / TikTok / YouTube / HN |
| **交互界面** | 非开发者友好 | Gradio Web UI (`localhost:7860`) |
| **零安装运行** | 云端笔记本 | Google Colab（计划中） |
| **容器化** | 一键部署 | Docker Compose 支持 |

**成本参考**：Premium 配置（Claude+Gemini+ElevenLabs）约 **$0.11/条**；Budget 配置约 **$0.04/条**；完全本地免费模式 **$0.00**。

## 二、可自定义的节点（高度模块化）

Verticals v3 的最大特色是 **Niche Profile** 系统，每个环节都可独立配置或单独执行：

### 1. Niche Profile（核心自定义）
这是 YAML 配置文件，决定整条管道的"人格"和"审美"。放在 `niches/` 目录下即可生效，**15 个内置领域可直接复制修改**：
- **tech**、gaming、finance、fitness、cooking、travel、true_crime、science、politics、entertainment、sports、fashion、education、motivation、comedy

每个 Profile 可自定义：
- **Script**：语气（tone）、节奏（pacing）、钩子模板（hooks）、禁用词（forbidden）、字数、CTA 变体
- **Visuals**：视觉风格（style）、情绪（mood）、主体（subjects）、避免元素（avoid）
- **Voice**：语速（pace）、能量感（energy）、推荐音色（按 TTS 提供商细分）
- **Captions**：高亮色（highlight_color）、字重（font_weight）、位置（position）
- **Music**：情绪（mood）、能量（energy）
- **Thumbnail**：风格、文字位置
- **Discovery**：各平台数据源配置（Reddit 子版块、RSS 源、Trends 分类等）

**自定义方法**：复制 `niches/tech.yaml` → 修改内容 → 重命名为 `niches/你的领域.yaml` → 执行时 `--niche 你的领域`。

### 2. Provider 层（可插拔服务商）
每个生产阶段都可独立切换提供商：

| 阶段 | 可选 Provider | 自定义方式 |
|------|--------------|-----------|
| **LLM（脚本）** | Claude / Gemini / GPT / **Ollama(本地)** | `--provider ollama` |
| **TTS（语音）** | Edge TTS / ElevenLabs / **Kokoro(本地)** / `say` | `--voice kokoro` |
| **Visuals（画面）** | Gemini / Replicate / Pexels / **ComfyUI(本地)** | `--visuals comfyui` |
| **上传平台** | YouTube / TikTok / Reels / X | `--platform youtube` |
| **语言** | en, hi, es, pt, de, fr, ja, ko 等 | `--lang ja` |

### 3. Stage 级独立执行（管道节点）
支持单步运行和断点续作，方便调试或替换中间素材：
```bash
python -m verticals draft --topic "话题" --niche tech     # 仅生成脚本
python -m verticals produce --draft <路径> --voice edge   # 仅生产视频
python -m verticals upload --draft <路径> --platform x    # 仅上传
python -m verticals topics --niche tech --limit 20        # 仅发现热点
```

### 4. 其他可配置项
- **API Keys**：`~/.verticals/config.json`（0600 权限），支持环境变量覆盖
- **音乐库**：项目 `music/` 目录内置免版税音乐，可替换或扩充
- **字幕样式**：ASS 模板可自定义字体、颜色、动画
- **Ken Burns 效果**：B-roll 画面的缩放/平移参数可在组装阶段调整

## 三、本地素材支持情况

**明确支持本地素材和本地运行**，且架构上为本地留有专门接口：

| 本地能力 | 说明 |
|---------|------|
| **本地 LLM** | 通过 **Ollama** 接入，无需 API Key，支持 7B+ 本地模型 |
| **本地 TTS** | 支持 **Kokoro**（开源本地 TTS）和 macOS 系统 `say` |
| **本地图像生成** | 支持 **ComfyUI** 作为 visuals provider，需本地运行 ComfyUI 服务 |
| **本地音乐** | 项目自带 `music/` 目录，可放入自有免版税音乐供 pipeline 调用 |
| **本地素材库** | **Pexels / Pixabay** 作为 stock footage fallback，完全免费（需免费 API Key） |

**关于"使用已有本地视频/图片素材"**：
Verticals v3 的默认流程是自动生成或拉取网络素材，但其架构基于模块化 stage（`stages/broll.py` 负责视觉素材）。如果你已有本地素材，可通过以下方式接入：
1. **替换 B-roll 目录**：在 `produce` 阶段前，将本地 9:16 素材放入 draft 工作目录的指定位置，跳过大模型生成
2. **ComfyUI 节点**：若使用 ComfyUI 作为 visuals provider，可在 ComfyUI 工作流中加载本地图片作为输入
3. **自定义 Stage**：由于代码完全开源，可修改 `stages/broll.py` 或 `stages/assemble.py` 直接读取本地素材路径

**一句话总结**：Verticals v3 是专为"AI 自动生成"设计的管道，但**所有节点均可本地替换或自定义**，如果你已有本地素材库，只需在 produce 阶段替换其 B-roll 输入即可接入，无需重写整个工具。
