# 任务恢复机制

> 长任务卡死后自动恢复进度

---

## 问题背景

OpenClaw等Agent执行长任务时：
- 任务卡死，无法继续
- 进度丢失，需要重新开始
- 中间产物浪费

---

## 状态持久化

### 状态文件位置

```
~/.verticals/media/work_{job_id}/state.json
```

### 状态文件内容

```json
{
  "job_id": "1778198693",
  "topic": "美军称袭击伊朗军事设施",
  "stage": "broll",
  "stages": {
    "draft": {"status": "completed", "output": "draft.json"},
    "voiceover": {"status": "completed", "output": "voiceover_zh.mp3"},
    "broll": {"status": "in_progress", "completed": 3, "total": 20},
    "captions": {"status": "pending"},
    "assemble": {"status": "pending"}
  },
  "created": "2026-05-08T08:05:00",
  "updated": "2026-05-08T08:07:30"
}
```

---

## 阶段定义

| 阶段 | 说明 | 输出 |
|------|------|------|
| draft | 生成脚本 | draft.json |
| voiceover | 生成语音 | voiceover_zh.mp3 |
| broll | 生成素材 | broll_*.png |
| captions | 生成字幕 | captions_zh.ass |
| assemble | 合成视频 | verticals_*.mp4 |

---

## 恢复流程

### 1. 检测未完成任务

启动时检查是否有未完成的 `work_*` 目录：

```bash
ls ~/.verticals/media/work_*/state.json
```

### 2. 显示进度摘要

```
发现未完成任务：
- 任务ID：1778198693
- 主题：美军称袭击伊朗军事设施
- 进度：素材生成中 (3/20)
- 上次更新：2026-05-08 08:07

请回复：
- "继续" - 继续上次任务
- "重新开始" - 删除进度，重新生成
- "取消" - 退出
```

### 3. 继续任务

从上次中断的阶段继续：

```
继续任务 1778198693...
从素材生成阶段恢复，已完成 3/20，继续生成剩余 17 张图片...
```

---

## 实现要点

### 每个阶段完成后

1. 更新状态文件
2. 记录输出文件路径
3. 标记阶段状态

### 阶段状态

| 状态 | 说明 |
|------|------|
| pending | 等待执行 |
| in_progress | 执行中 |
| completed | 已完成 |
| failed | 失败，需重试 |

### 恢复逻辑

```python
def resume_task(job_id):
    state = load_state(job_id)
    for stage_name, stage_info in state["stages"].items():
        if stage_info["status"] == "completed":
            continue  # 跳过已完成
        if stage_info["status"] == "in_progress":
            # 从中断处继续
            resume_stage(stage_name, stage_info)
            break
        if stage_info["status"] == "pending":
            # 执行未开始的阶段
            run_stage(stage_name)
```

## 心跳机制

长任务执行时，定期输出心跳：

```
[素材生成] 正在生成第 5/20 张图片... (预计剩余 2分钟)
[素材生成] 正在生成第 6/20 张图片... (预计剩余 1分50秒)
```

心跳间隔：每完成一个子任务输出一次

---

## 注意事项

1. 状态文件每个阶段完成后立即保存
2. 恢复时检查中间产物是否完整
3. 失败阶段最多重试3次
4. 超过24小时的任务标记为过期
