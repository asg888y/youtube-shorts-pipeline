# 质量标准与良品率

> 音频长度是视频完整度的唯一标尺

---

## 核心原则

**视频时长 = 音频时长**

所有素材必须填满音频时长，不能多也不能少。

---

## 素材数量计算

### 公式

```
素材数量 = ceil(音频时长 / 单个素材展示时长)
```

### 示例

| 音频时长 | 切换秒数 | 素材数量 |
|---------|---------|---------|
| 30秒 | 2秒 | 15个 |
| 30秒 | 3秒 | 10个 |
| 45秒 | 2秒 | 23个（向上取整）|
| 60秒 | 2.5秒 | 24个 |

### 计算逻辑

```python
import math
素材数量 = math.ceil(音频时长 / switch_seconds)
```

---

## 素材类型与成本

| 类型 | 单价 | 时长限制 | 说明 |
|------|------|----------|------|
| 图片 | $0.05/张 | 由 switch_seconds 决定 | Ken Burns动画展示 |
| 视频 | $0.2/个 | 4/6/8秒 | RunningHub限制 |

### 视频素材特殊规则

1. RunningHub视频API仅支持 **4/6/8秒** 三种时长
2. 视频生成后会被截取/循环到 `switch_seconds` 时长
3. 视频成本是图片的 **4倍**，谨慎使用

---

## 良品率检查清单

生成前必须确认：

| 检查项 | 说明 |
|--------|------|
| ✅ 音频时长 | 先有音频，才知道需要多少素材 |
| ✅ 素材数量 | `素材数 = ceil(音频时长 / 切换秒数)` |
| ✅ 成本预估 | 图片数×$0.05 + 视频数×$0.2 |
| ✅ Prompt质量 | 每个素材需要独立prompt，禁止重复 |

---

## 常见废片原因

| 问题 | 原因 | 解决 |
|------|------|------|
| 视频太短 | 素材不够填满音频 | 增加素材数量 |
| 视频太长 | 素材超过音频时长 | 减少素材数量 |
| 画面重复 | prompt相同或相似 | 每个prompt差异化 |
| 纯色画面 | API失败，fallback填充 | 检查API状态，重试 |
| 字幕不同步 | Whisper识别错误 | 检查音频质量 |
| 画面黑边 | 素材比例不对 | 使用9:16竖屏素材 |

---

## Prompt质量要求

### 禁止

- ❌ 使用相同prompt生成多张图片
- ❌ 使用模板复制prompt
- ❌ prompt过于简单（如"风景"）

### 推荐

- ✅ 每个prompt独立描述场景
- ✅ 包含：场景、色调、光线、氛围
- ✅ 与文案内容相关联

### 示例

```
好的prompt：
"伊朗军事基地夜景，导弹发射井，红色警报灯光，紧张氛围，电影级画质"

差的prompt：
"军事基地"
```

## 质检门禁

生成完成后自动检查：

| 门禁 | 检查内容 | 不通过则 |
|------|----------|---------|
| G1 | 音频时长 > 0 | 停止 |
| G2 | 素材数量 >= 计算值 | 停止 |
| G3 | BGM存在 | 停止 |
| G4 | 字幕文件存在 | 警告 |
| G5 | 视频可播放 | 停止 |
